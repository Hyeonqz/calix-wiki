# 컨텍스트 관리

Claude가 한 번에 처리할 수 있는 대화량은 유한하다 — **컨텍스트 윈도우**. 사이즈가 정해진 작업 노트 같은 것. 다 차면 새 노트로 시작하거나, 핵심만 남기고 정리해야 한다.

## `/clear` vs `/compact`

| | `/clear` | `/compact` |
|---|---|---|
| 효과 | 대화 이력 **완전 초기화** | 핵심만 남기고 **압축** |
| 시점 | **새 작업** 시작 시 | **같은 작업** 이어갈 때 |
| 비유 | 노트를 지우고 새로 시작 | 핵심만 남기고 공간 확보 |

`/clear` 후에는 이전 대화를 모르므로 *"아까 그 파일"* 같은 표현 금지. 이건 **새 세션이 아니라** 같은 세션 안에서 노트만 비우는 것. 완전히 새 세션을 원하면 `/exit` → `claude` 다시 실행.

## `/compact` — 유지할 내용 지정

키워드를 뒤에 붙이면 더 정확하게 압축된다.

```
/compact API 변경사항에 집중해줘
/compact 수정한 파일 목록과 남은 작업을 유지해줘
```

압축 시 살아남는 것 / 사라지는 것:

| 디스크에서 재주입 | 요약에 포함 | 사라짐 |
|---|---|---|
| 시스템 프롬프트 | 사용자 요청과 의도 | 도구 출력 전체 (파일 내용, 명령 결과) |
| CLAUDE.md (글로벌+프로젝트) | 검토/수정한 파일과 핵심 코드 조각 | 중간 추론 과정 |
| Auto Memory | 에러와 해결 방법 | 호출 안 한 스킬 설명 |
| 환경 정보, MCP 도구 목록 | 대기 중 작업과 현재 상태 | |

**핵심**: 대화에서만 한 지시(*"한국어로 응답해줘"* 등)는 압축 시 사라질 수 있다. 반복 규칙은 [CLAUDE.md](/claude-code/claude-md-best-practices)에 적어두면 압축 후에도 유실되지 않는다.

## `/context` — 사용량 확인

현재 컨텍스트가 얼마나 차 있는지, 카테고리별로 어디에 토큰이 쓰였는지 표시한다.

| 항목 | 내용 | 로드 시점 |
|---|---|---|
| System prompt | Claude의 기본 동작 지시 | 세션 시작 |
| System tools | 도구 정의 | 세션 시작 |
| Memory files | CLAUDE.md, Auto Memory | 세션 시작 |
| Skills | 스킬 이름·설명 (전체 내용은 호출 시 로드) | 세션 시작 |
| MCP servers | 연결된 MCP 서버의 도구 정의/스키마 | 세션 시작 |
| Messages | 대화, 파일 내용, 명령 결과 | 대화 중 |

대화를 시작하기도 전에 시스템 프롬프트·도구·CLAUDE.md·MCP 서버가 이미 공간을 차지한다. CLAUDE.md가 길거나 MCP 서버가 많으면 기본 사용량이 크다. **`/mcp`로 서버별 비용 확인 → 안 쓰는 서버 끊기**가 가장 빠른 절감법.

## 자동 압축

컨텍스트의 **약 95%**가 차면 자동 압축이 실행된다. 자동 압축도 토큰을 소비하므로, **50% 정도에서 미리 `/compact`**가 비용·품질 모두 유리하다. 컨텍스트가 작을수록 응답 품질도 좋다.

수동 `/compact`는 유지할 내용을 직접 지정할 수 있어 자동보다 정확하다.

## 압축 시점 환경변수

```bash
# 50%에서 자동 압축 (기본 95%)
export CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=50

# 1M 모델에서 500K 윈도우로 압축 (나머지 500K 비용 절감)
export CLAUDE_CODE_AUTO_COMPACT_WINDOW=500000
```

`PCT_OVERRIDE`는 위 윈도우의 백분율로 적용. 두 변수를 같이 쓰면 250K에서 압축.

## 실용 팁

- **70% 넘으면** `/compact` 고려
- **같은 문제를 두 번 이상 수정**했다면 → 실패한 시도들이 컨텍스트를 어지럽히는 중. `/clear` 후 배운 것을 새 프롬프트에 반영해 다시 시작하는 게 더 효과적
- **반복 규칙**은 대화에 적지 말고 [CLAUDE.md](/claude-code/claude-md-best-practices)에
