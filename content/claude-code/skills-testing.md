# Skills 테스트

스킬 테스트는 엄격함의 수준에 따라 세 가지 방법이 있다:

| 방법 | 특징 | 적합한 경우 |
|---|---|---|
| Claude.ai에서 수동 테스트 | 빠른 반복, 설정 불필요 | 초기 개발, 개인 스킬 |
| Claude Code에서 스크립트 테스트 | 반복 가능한 검증, 테스트 케이스 자동화 | 팀 스킬, 품질 보증 |
| 스킬 API를 통한 프로그래밍 테스트 | 체계적 평가 스위트 | 기업용, 대규모 배포 |

> **Pro Tip:** 가장 효과적인 스킬 창작자는 단일 도전적인 작업에서 먼저 반복하고, 성공적인 접근 방식을 스킬로 추출한다. 작동하는 기반이 생기면 여러 테스트 케이스로 확장.

## 세 가지 테스트 영역

### 1. 트리거 테스트

스킬이 올바른 시간에 로드되는지 확인:

```
트리거되어야 함:
- "새 ProjectHub 워크스페이스 설정 도와줘"
- "ProjectHub에서 프로젝트를 생성해야 해"

트리거되지 않아야 함:
- "샌프란시스코 날씨는 어때?"
- "Python 코드 작성 도와줘"
```

### 2. 기능 테스트

스킬이 올바른 출력을 생성하는지 확인:

```
테스트: 5개 작업으로 프로젝트 생성
Given: 프로젝트 이름 "Q4 계획", 5개 작업 설명
When: 스킬이 워크플로를 실행할 때
Then:
  - 프로젝트 생성됨
  - 올바른 속성으로 5개 작업 생성됨
  - 모든 작업이 프로젝트에 연결됨
  - API 오류 없음
```

### 3. 성능 비교

스킬이 기준선 대비 결과를 개선함을 증명:

| 항목 | 스킬 없이 | 스킬과 함께 |
|---|---|---|
| 워크플로 | 매번 사용자가 명령어 제공 | 자동 실행 |
| 왕복 메시지 | 15회 | 2회 (명확화만) |
| 실패한 API 호출 | 3회 (재시도 필요) | 0회 |
| 토큰 소비 | 12,000 | 6,000 |

## skill-creator 사용

`skill-creator` 스킬은 Claude.ai(플러그인 디렉터리)와 Claude Code에서 사용 가능하며, 15-30분 내에 기능하는 스킬을 구축할 수 있다.

### 기능

| 기능 | 설명 |
|---|---|
| 스킬 생성 | 자연어 설명에서 적절한 SKILL.md 생성, 트리거 문구 제안 |
| 스킬 검토 | 모호한 설명/트리거 누락/구조 문제 표시, 과도/부족 트리거 위험 식별 |
| 반복적 개선 | 엣지 케이스 해결, 테스트 케이스 제안 |

**사용법:**
```
"[사용 사례]를 위한 스킬 구축을 skill-creator 스킬을 사용하여 도와줘"
```

> skill-creator는 설계 및 개선을 돕지만, 자동화된 테스트 스위트를 실행하거나 정량적 평가를 생성하지는 않는다.

## 피드백 기반 반복

스킬은 살아있는 문서다:

### 부족 트리거 (스킬이 로드되어야 할 때 안 됨)

- **신호:** 사용자가 수동으로 활성화, "언제 사용하나요?" 질문
- **해결:** description에 더 많은 세부 정보와 키워드 추가

### 과도 트리거 (관련 없는 쿼리에 로드됨)

- **신호:** 사용자가 비활성화, 목적에 대한 혼란
- **해결:** 부정적 트리거 추가, 더 구체적으로 만들기

```yaml
# 부정적 트리거 예시
description: CSV 파일에 대한 고급 데이터 분석. 통계 모델링에 사용합니다.
  간단한 데이터 탐색에는 사용하지 마세요 (대신 data-viz 스킬 사용).
```

### 실행 문제 (일관성 없는 결과)

- **신호:** API 호출 실패, 사용자 수정 필요
- **해결:** 명령어 개선, 오류 처리 추가
