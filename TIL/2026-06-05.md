## 배운 것

| # | 내용 |
|---|------|
| 1 | Java는 **패키지 선언과 실제 디렉토리 경로가 반드시 일치**해야 컴파일된다. 파일이 `application/services/` 디렉토리에 있으면 `package ... application.services;`를 선언해야 한다 |
| 2 | Mockito `@InjectMocks`는 **생성자 주입 방식**으로 Mock을 주입한다. 생성자 파라미터 중 하나라도 `@Mock`이 없으면 해당 의존성이 `null`로 주입돼 NPE 발생 가능 |
| 3 | **상속 구조**(`extends BaseService`)에서 `super(...)` 생성자 파라미터도 모두 `@Mock`으로 등록해야 `@InjectMocks`가 정상 동작한다 (`CresotyBaseService` 6개 + `CresotyTransactionService` 7개 = 총 13개) |
| 4 | `BDDMockito.then(...).should(never())`: 특정 메서드가 **한 번도 호출되지 않음**을 검증하는 패턴 |

| 8 | `FieldType` 의미 — `N`(숫자만), `AN`(영문+숫자), `ANS`(영문+숫자+특수문자): UUID나 승인번호처럼 하이픈·특수기호 포함 필드는 `ANS` 선언 필수 |
| 9 | Swagger `@Parameter(example = ...)` 는 정적 값만 가능 — 오늘 날짜처럼 동적 값은 `OperationCustomizer` 빈을 `@Configuration`에서 선언해 `operation.getParameters()`를 순회하여 런타임에 주입해야 함 |
| 10 | 스펙 문서와 코드의 FieldType/Length 불일치는 패딩 계산 오류로 이어지므로, 최초 구현 시 PDF 스펙을 필드별로 전수 비교하는 것이 필수 |
| 11 | IntelliJ "Variable initializer is redundant" 경고 — 초기값이 실제로 **읽히는 경로가 없을 때** 발생. catch에서 항상 throw하면 그 이후 코드에서 값이 항상 할당됨을 컴파일러가 인지해 `= ""`나 `= null` 초기화가 불필요해짐 |
| 12 | `CurrencyUtil.makeAmountWithCurrency()` 결과는 이미 `String`이라 `Field<BigDecimal>`의 `formatField()`를 통과시킬 수 없음 → `formatAmount(currencyCode, field)` 헬퍼 패턴으로 해결: `CurrencyUtil` 호출 후 `StringUtils.leftPad(str, field.getLength(), '0')` 적용 |
