## 배운 것

| # | 내용 |
|---|------|
| 1 | Tomcat 로그에서 `IllegalArgumentException: Invalid character found in the request target [/public/index.php?s=/index/\think\app/...]`는 ThinkPHP RCE 취약점 스캔 봇이 보내는 공격 시도다. Tomcat이 RFC 7230 기준 백슬래시를 파싱 단계에서 차단하므로 애플리케이션 코드에는 도달하지 않는다. 조치 불필요 |
| 2 | Tomcat 파싱 오류 로그를 억제하려면 `logging.level.org.apache.coyote.http11: WARN` 설정으로 INFO 레벨 스택트레이스를 숨길 수 있다. 최초 1회 이후 Tomcat이 자체적으로 DEBUG로 격하한다 |
| 3 | `supports/web` 같은 공유 라이브러리 모듈에 필터를 두면 `@SpringBootApplication`의 컴포넌트 스캔 범위 밖이라 `@Component`만으로는 자동 등록되지 않는다. `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`에 `@AutoConfiguration` 클래스를 등록해야 소비 모듈이 의존성 추가만으로 필터를 자동으로 가져간다 |
| 4 | `WebSecurityConfig.addFilterBefore(filter, SecurityContextHolderFilter.class)`는 필터를 Spring Security 체인 내부에 등록한다. Auto-Configuration `@Bean`으로 등록하면 Servlet 필터 체인(Spring Security 앞)에 등록되어 더 앞단에서 차단할 수 있다 |
| 5 | `OncePerRequestFilter`를 두 곳에 중복 등록하면(Security 체인 + Servlet 필터) `OncePerRequestFilter`의 요청 속성 기반 중복 방지 메커니즘으로 한 번만 실행되지만, 코드 명확성을 위해 하나만 유지하는 것이 맞다 |
| 6 | Gradle의 `ChecksumService` I/O 오류(`java.io.IOException: Input/output error at LockStateAccess.readState`)는 WSL2에서 Windows NTFS 경로(`/mnt/c/...`)의 `.gradle/8.5/checksums/checksums.lock` 파일에 대한 POSIX 파일 락이 실패하는 현상이다. Windows 측(PowerShell 또는 탐색기)에서 해당 파일을 삭제하거나 WSL을 재시작하면 해결된다 |
| 7 | enum 파라미터로 분기 설계 시, 동일한 메서드를 여러 경로에서 호출하는 경우 파라미터로 출처를 명시하면 로그 추적과 향후 분기 처리가 용이해진다. 단순 분기는 `switch`가 아닌 로그 출력만으로도 충분히 가치 있다 |
| 8 | Cresoty 배치에서 빈 데이터 early return 시 파일을 아예 생성하지 않으면 외부 수신측에서 파일 미수신으로 오류가 날 수 있다. 데이터 유무와 무관하게 헤더+푸터만 있는 빈 파일을 항상 업로드하는 패턴이 외부 연동 배치의 안전한 설계다 |
| 9 | 배치에서 빈 리스트를 그대로 DB 쿼리(`IN ()`)에 전달하면 일부 DB에서 오류가 날 수 있다. early return 구조를 유지하면서 빈 파일 처리를 별도 메서드로 분리하면 DB 쿼리 안전성과 파일 생성 책임을 동시에 만족시킬 수 있다 |
| 10 | JPA `@OneToOne(fetch = FetchType.LAZY)` 관계를 stream 필터에서 직접 호출하면 N+1이 발생한다. QueryDSL `join(...).fetchJoin()`으로 연관 엔티티를 즉시 로딩하면 단일 쿼리로 해결된다. 이후 `h.getPayment().getXxx()` 호출 시 추가 SELECT 없음 |
| 11 | QueryDSL `fetchJoin()`을 사용하면 `where` 절에 JOIN 대상 엔티티의 컬럼 조건(`qPayment.partnerCode.eq(...)`)을 직접 걸 수 있다. 이를 통해 서비스 레이어에서 stream 필터 없이 DB에서 필터링된 데이터를 바로 가져올 수 있다 |
